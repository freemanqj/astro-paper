# Top 6 In-Demand Skills for Product Managers

正文写在这里…


プロダクトのゼロから 1 への立ち上げから、継続的なイテレーションと進化へと至るプロセスは、複数の役割が連携するシステム工学です。開発チームだけでなく、プロダクトマネージャー、オペレーション、データアナリスト、UI/UX デザイナーといった職種が深く関与します。

TRAE ユーザーとの日常的なやり取りの中で、多くのプロダクトマネージャーが業務ワークフローに TRAE IDE を積極的に取り込んでいることを実感しています。

こうした背景から、プロダクトマネージャーの業務シーンに特化した人気スキル 6 選を厳選しました。企画立案から設計、実行計画の策定、リリース後のデータ追跡まで、全工程をカバーすることで、各重要段階での業務効率を飛躍的に高めることを目指しています。

### アイデア構想ブレインストーミング

**名称**

brainstorming

**作者**

obra（superpowers）

**URL**

[https://github.com/obra/superpowers](https://github.com/obra/superpowers)

**説明**

このスキルは、曖昧なアイデアを構造化された対話によって、明確な設計案へと落とし込むことができます。

「一度に一つの質問」を投げかけるスタイルで、まるでベテランのプロダクトコンサルタントのように、ニーズを段階的に明確にしていきます。自動的に実現可能な案を 2～3 つ提示し、理由を添えて推奨案を示した上で、最終的にアーキテクチャ、データフロー、テスト戦略まで網羅した完全な設計ドキュメントを出力します。

**インストールコマンド**
npx skills add obra/superpowers --skill brainstorming

**活用シーン**

- 新機能コンセプト審査：PRD 作成前に、対話を通じてアイデアを広げ、最適な案に絞り込む
- 業務プロセス再設計：複雑な業務ロジックを整理し、多角的な視点から改善策を提示
- 不確実性の高い革新プロジェクト：「方向性はあるが具体化できない」という場面に最適
- 内部審査準備：AI で事前に審査の質問をシミュレートし、設計の漏れや矛盾点を洗い出す

**使用例**

plaintext

```
「AIライティングアシスタントを作りたいので、プロダクト形態をブレインストーミングしてください」
「ユーザーアクティベーション率が低いので、初心者ガイドの設計案を考えてください」
「この機能の3つの実装案を分析し、推奨案を提示してください」
```

### 要件設計

**名称**

write-a-prd

**作者**

mattpocock

**URL**

[https://github.com/mattpocock/skills](https://github.com/mattpocock/skills)

**説明**

プロダクトマネージャー向けに特化した PRD（製品要求仕様書）作成スキルです。単なるテンプレート入力ではなく、対話形式のインタビューを通じて要件の全体像を丁寧に整理し、GitHub Issue としてそのまま利用可能な、構造化された高品質な PRD を自動生成します。

「インタビュー形式」のワークフローを採用しており、課題や実現したいアイデアを詳しくヒアリングした上で、プロダクトレビューのように設計の分岐点を一つひとつ確認し、共通理解を深めたうえで最終的な PRD を作成します。

**PRD の構成**

- ユーザー目線での課題説明
- ユーザー目線での解決策説明
- 詳細なユーザーストーリー一覧
- モジュール分割、インターフェース設計、アーキテクチャ情報
- テスト範囲・推奨テスト戦略
- その他補足事項

**インストールコマンド**

plaintext

```
npx skills add mattpocock/skills --skill write-a-prd
```

**活用シーン**

- 新機能の要件作成：曖昧なアイデアから、対話を通じて実行可能な PRD を構築
- レビュー前の確認：AI がレビュアー役となり、論理の矛盾や漏れを事前に洗い出し
- 部門間コミュニケーション：ユーザーストーリーを軸に、開発・デザインチームと価値を共有
- イテレーション記録：機能ごとに追跡可能な履歴を残す

**使用例**

plaintext

```
「ポイント・バッジ・特典を含むユーザーランクシステムを作成したいのでPRDを作成してください」
「この機能アイデアをPRDにまとめてください。解決したい課題は『ユーザーが自身の成長を実感できないこと』です」
```

### 実行計画の分解

**名称**

writing-plans

**作者**

obra（superpowers）

**URL**

[https://github.com/obra/superpowers](https://github.com/obra/superpowers)

**説明**

要件設計を、極めて細かい単位の実行計画に分解してくれるスキルです。

各ステップは 2～5 分で完了する規模で、明確なファイルパス、実行コマンド、そして验收（動作確認）の基準がセットで記載されます。元々は開発実行用に設計されていますが、「複雑な目標を、验收基準付きのマイクロタスク一覧に分解する」という核心的な能力は、プロダクトマネージャーがプロジェクトの実行方針を立てたり、コンテンツ公開計画を作成したり、開発チームと調整する際の細かいタスク分解資料としても非常に役立ちます。

各計画書には以下が含まれます：機能目標（1 文）、アーキテクチャ説明（2～3 文）、技術スタック、ファイル構成マップ、各タスクブロックのステップ別チェックリスト（コード、コマンド、期待される結果付き）。

**インストールコマンド**

plaintext

```
npx skills add obra/superpowers --skill writing-plans
```

**活用シーン**

- プロジェクト分解：大きなプロダクト要件を、追跡可能な順序立てた実行サブタスクに分解
- コンテンツ公開計画：各ノードでの成果物、担当者、验收基準を明確化
- 開発チームとの調整：開発視点の実行計画草案を作成し、工数評価の参考資料とする
- brainstorming スキルとの連携：設計案確定後、自動的に実行計画を生成

**使用例**

plaintext

```
「ユーザー成長プラットフォームの設計案を、実行可能な複数の計画に分解してください」
```

### A/B テスト設計

**名称**

ab-test-setup

**作者**

coreyhaines31

**URL**

[https://github.com/coreyhaines31/marketingskills](https://github.com/coreyhaines31/marketingskills)

**説明**

プロダクトマネージャー・グロースチーム向けの**プロフェッショナルな A/B テスト設計スキル**です。統計的に厳密で、意思決定に直接活かせる実験を設計できます。仮説設計・必要サンプル数の計算・指標選定から、実験実行のルール、結果の正しい解釈まで全工程をサポート。「疑似実験」「途中で早めに止めてしまう失敗」など、よくある落とし穴を回避できます。

**インストールコマンド**

plaintext

```
npx skills add coreyhaines31/marketingskills --skill ab-test-setup
```

**活用シーン**

- ページ CVR 最適化：ランディングページ・登録フロー・決済ページの A/B テスト設計
- 新機能ロールアウト：リリース前の比較実験でユーザー行動の変化を定量評価
- コピー・ビジュアル検証：タイトル・CTA・画像などクリエイティブの効果測定
- 価格表示テスト：異なる価格の見せ方によるコンバージョン比較

**使用例**

plaintext

```
「登録ボタンを青からオレンジに変更するA/Bテストを設計してください」
「料金ページのCVRが3%です。新しい価値訴求をテストする場合、必要なサンプル数は？」
```

plaintext

```
npx skills add coreyhaines31/marketingskills --skill analytics-tracking

- オンボーディングファネル分析：各活性化ステップの转化率を定量的に計測

```

# 新規ユーザー活性化(onboarding‑cro)

作成者：coreyhaines31

リポジトリ：[https://github.com/coreyhaines31/marketingskills](https://github.com/coreyhaines31/marketingskills)

新規ユーザー活性化の全体最適化に特化したスキルです。

プロダクトマネージャーがユーザーのアハ体験を発見し、最初の成功体験にスムーズに到達できるオンボーディングフローを設計し、

複数チャネルによる活性化運用体制を構築できるよう支援します。

## インストール

plaintext

```
npx skills add coreyhaines31/marketingskills --skill onboarding-cro
```

## 活用シーン

- 活性化率向上：オンボーディングファネルを分析し、最大の離脱ポイントを特定
- 空ステート設計：空ページを活性化への導入機会に変換
- オンボーディングメール設計：登録後のタイミング（24h / 72h / 7 日目）に合わせたトリガーメールを設計
- 活性化実験設計：A/B テストを実施すべきオンボーディングステップを見極め

## 使用例

「定着ユーザーは、離脱ユーザーには見られないどのような行動をとっているのか？」
